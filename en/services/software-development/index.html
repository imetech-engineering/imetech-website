FAQ

Frequently asked questions

Do you work with existing embedded firmware?

Yes, that is a common assignment. I work with existing codebases on ESP32, STM32 and Arduino: adding features, refactoring for maintainability or debugging stubborn issues. I can also take over or improve PLC programs in Structured Text.

Which platforms and programming languages do you support?

I develop embedded software in C/C++ for ESP32, STM32 and Arduino, plus Python for backends and data processing. For industrial automation I write PLC logic for WAGO and Beckhoff, including Modbus and Profinet integrations.

What does a software project with IMeTech look like?

We start with a discovery call and requirements check. Then comes a proof-of-concept or prototype, followed by iterative delivery with clear milestones. This keeps scope, timeline and budget under control throughout the project.

Do you write PLC logic for industrial automation?

Yes. I develop control logic in Structured Text for WAGO and Beckhoff PLCs, including process automation, fieldbus communication and integration with existing machinery. I often combine this with embedded firmware and Python scripts for data processing.

Do you work remotely on embedded projects?

Yes, most software and embedded projects run partly or fully remote via video calls, shared repositories and clear documentation. Based in Aalten in the Achterhoek, I work with clients across the Netherlands; on-site visits are possible when needed.
